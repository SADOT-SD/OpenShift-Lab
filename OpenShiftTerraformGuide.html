OpenShift 4.x on VMware ESXi 8 via Terraform (UPI)

This comprehensive guide details how to provision an OpenShift 4.x cluster on VMware vSphere using the User-Provisioned Infrastructure (UPI) method with Terraform. It includes Prerequisites (DNS & LB), Shared Modules, Standard/Compact cluster configurations, and access instructions.

PART 0: Prerequisites - DNS & Load Balancer

Before running Terraform, your network must be prepared to route traffic to the OpenShift cluster. OpenShift strictly relies on DNS resolution and an external Load Balancer (like HAProxy, F5, or NGINX) in a UPI deployment.

0.1 Load Balancer Configuration

Your Load Balancer must listen on specific ports and forward traffic to the corresponding nodes. Note: The Bootstrap node should only be included in the LB pools during the installation phase. Once the cluster is up, it is removed.

Port Protocol Target Nodes (Backend Pool) Purpose
6443 TCP Bootstrap, Masters Kubernetes API Server
22623 TCP Bootstrap, Masters Machine Config Server (Ignition delivery)
80 TCP Workers (or Masters if Compact) HTTP Application Ingress
443 TCP Workers (or Masters if Compact) HTTPS Application Ingress
0.2 DNS Records Configuration

You must configure the following A-records in your DNS server. All these records should point to the IP address of your Load Balancer.

  • api.<cluster_name>.<base_domain> (Points to LB IP, handles port 6443)
  • api-int.<cluster_name>.<base_domain> (Points to LB IP, handles ports 6443 and 22623)
  • *.apps.<cluster_name>.<base_domain> (Wildcard pointing to LB IP, handles ports 80/443)
PART 1: Shared Configuration & Modules

These files establish the foundation of your deployment, regardless of the cluster size.

Step 1: Project Structure & Provider Configuration

Create a directory structure that separates your modules from your root configuration:

ocp4-vsphere-upi/
├── providers.tf       
├── variables.tf       
├── terraform.tfvars   
├── main.tf            
└── modules/
    └── ocp_node/      
        ├── main.tf
        └── variables.tf

providers.tf

terraform {
  required_providers {
    vsphere = {
      source  = "hashicorp/vsphere"
      version = "~> 2.15"
    }
  }
}

provider "vsphere" {
  user                 = var.vsphere_user
  password             = var.vsphere_password
  vsphere_server       = var.vsphere_server
  allow_unverified_ssl = true 
}
Step 2: The Reusable Node Module

modules/ocp_node/variables.tf

variable "vm_name" { type = string }
variable "cpu" { type = number, default = 4 }
variable "ram_mb" { type = number, default = 16384 }
variable "disk_size_gb" { type = number, default = 120 }
variable "resource_pool_id" { type = string }
variable "datastore_id" { type = string }
variable "folder_id" { type = string }
variable "network_id" { type = string }
variable "rhcos_template_uuid" { type = string }
variable "ignition_data" { type = string }
variable "mac_address" { type = string, default = "" }

modules/ocp_node/main.tf

resource "vsphere_virtual_machine" "node" {
  name             = var.vm_name
  resource_pool_id = var.resource_pool_id
  datastore_id     = var.datastore_id
  folder           = var.folder_id

  num_cpus = var.cpu
  memory   = var.ram_mb
  guest_id = "coreos64Guest"

  network_interface {
    network_id   = var.network_id
    # Uncomment if using static MAC addresses:
    # use_static_mac = var.mac_address != "" ? true : false
    # mac_address    = var.mac_address
  }

  disk {
    label            = "disk0"
    size             = var.disk_size_gb
    thin_provisioned = true
  }

  clone {
    template_uuid = var.rhcos_template_uuid
  }

  extra_config = {
    "guestinfo.ignition.config.data"          = var.ignition_data
    "guestinfo.ignition.config.data.encoding" = "base64"
  }
}

A standard topology consisting of 1 Bootstrap, 3 Control Plane (Master) nodes, and 2 or more Worker nodes.

Standard & Compact main.tf
This `main.tf` logic is used for BOTH Standard and Compact clusters. It uses `count` loops based on the variables.
# 1. Fetch vSphere Data Sources
data "vsphere_datacenter" "dc" { name = var.datacenter_name }
data "vsphere_compute_cluster" "cluster" { name = var.cluster_name, datacenter_id = data.vsphere_datacenter.dc.id }
data "vsphere_datastore" "datastore" { name = var.datastore_name, datacenter_id = data.vsphere_datacenter.dc.id }
data "vsphere_network" "network" { name = var.network_name, datacenter_id = data.vsphere_datacenter.dc.id }
data "vsphere_virtual_machine" "rhcos_template" { name = var.rhcos_template_name, datacenter_id = data.vsphere_datacenter.dc.id }

# 2. Create a vSphere Folder
resource "vsphere_folder" "folder" {
  path          = var.cluster_id
  type          = "vm"
  datacenter_id = data.vsphere_datacenter.dc.id
}

# 3. Read Ignition Files
data "local_file" "bootstrap_ign" { filename = "${path.module}/install_dir/bootstrap.ign" }
data "local_file" "master_ign"    { filename = "${path.module}/install_dir/master.ign" }
data "local_file" "worker_ign"    { filename = "${path.module}/install_dir/worker.ign" }

# 4. Bootstrap Node
module "bootstrap" {
  source              = "./modules/ocp_node"
  vm_name             = "${var.cluster_id}-bootstrap"
  cpu                 = var.bootstrap_cpu
  ram_mb              = var.bootstrap_ram
  disk_size_gb        = var.disk_size
  resource_pool_id    = data.vsphere_compute_cluster.cluster.resource_pool_id
  datastore_id        = data.vsphere_datastore.datastore.id
  folder_id           = vsphere_folder.folder.id
  network_id          = data.vsphere_network.network.id
  rhcos_template_uuid = data.vsphere_virtual_machine.rhcos_template.id
  ignition_data       = base64encode(data.local_file.bootstrap_ign.content)
}

# 5. Control Plane (Master) Nodes
module "control_plane" {
  source              = "./modules/ocp_node"
  count               = var.master_count
  vm_name             = "${var.cluster_id}-master-${count.index}"
  cpu                 = var.master_cpu
  ram_mb              = var.master_ram
  disk_size_gb        = var.disk_size
  resource_pool_id    = data.vsphere_compute_cluster.cluster.resource_pool_id
  datastore_id        = data.vsphere_datastore.datastore.id
  folder_id           = vsphere_folder.folder.id
  network_id          = data.vsphere_network.network.id
  rhcos_template_uuid = data.vsphere_virtual_machine.rhcos_template.id
  ignition_data       = base64encode(data.local_file.master_ign.content)
}

# 6. Worker Nodes
module "workers" {
  source              = "./modules/ocp_node"
  count               = var.worker_count
  vm_name             = "${var.cluster_id}-worker-${count.index}"
  cpu                 = var.worker_cpu
  ram_mb              = var.worker_ram
  disk_size_gb        = var.disk_size
  resource_pool_id    = data.vsphere_compute_cluster.cluster.resource_pool_id
  datastore_id        = data.vsphere_datastore.datastore.id
  folder_id           = vsphere_folder.folder.id
  network_id          = data.vsphere_network.network.id
  rhcos_template_uuid = data.vsphere_virtual_machine.rhcos_template.id
  ignition_data       = base64encode(data.local_file.worker_ign.content)
}

Compact install-config.yaml Note
Crucial Step: For a compact cluster, modify your install-config.yaml BEFORE generating the Ignition files so the Masters can run workloads: 
compute:
- name: worker
  replicas: 0
controlPlane:
  name: master
  replicas: 3
PART 4: Execution & Validation
Step 5: Plan and Apply

Ensure your Ignition files are in the install_dir/ directory.

  1. Run terraform init
  2. Run terraform plan
  3. Run terraform apply -auto-approve

Once installation completes, remove the Bootstrap node from Terraform and your Load Balancer.

PART 5: Accessing Your Cluster
Step 6: Accessing via CLI and GUI

1. Command Line Interface (CLI) Access

export KUBECONFIG=/path/to/your/install_dir/auth/kubeconfig
oc get nodes
oc get clusteroperators

2. Graphical User Interface (GUI) Access

  • Console URL: https://console-openshift-console.apps.<cluster_name>.<base_domain>
  • Username: kubeadmin
  • Password:
cat /path/to/your/install_dir/auth/kubeadmin-password
