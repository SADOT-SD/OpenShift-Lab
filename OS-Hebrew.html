מדריך התקנת OpenShift על AWS (IPI) - גרסת Linux Terminal

מדריך זה מיועד להתקנת Red Hat OpenShift על גבי תשתית AWS באמצעות שיטת Installer-Provisioned Infrastructure (IPI), מתוך סביבת עבודה מבוססת Linux לחלוטין.

דרישות קדם:
  • גישת Terminal / SSH למכונת Linux (למשל Ubuntu, RHEL או Amazon Linux).
  • חשבון AWS פעיל עם הרשאות Administrator.
  • דומיין (Hosted Zone) מוגדר ב-AWS Route 53.
  • חשבון פעיל ב-Red Hat עם Pull Secret זמין.

שלב 1: הגדרת אישורי AWS (AWS Credentials)

הכלי של OpenShift משתמש ב-AWS CLI כדי לתקשר עם הענן. הגדר את משתני הסביבה (או הגדר את ~/.aws/credentials):

export AWS_ACCESS_KEY_ID="YOUR_ACCESS_KEY"
export AWS_SECRET_ACCESS_KEY="YOUR_SECRET_KEY"
export AWS_DEFAULT_REGION="us-east-1" # שנה לאזור המבוקש

שלב 2: הורדת כלי ההתקנה (openshift-install ו-oc)

הורד את הכלים הייעודיים ללינוקס ישירות מהאתר של Red Hat וחלץ אותם לתיקייה מקומית:

# יצירת תיקיית עבודה
mkdir ~/openshift-install-dir
cd ~/openshift-install-dir

# הורדת openshift-install (יש לוודא שימוש בגרסה העדכנית ביותר הנדרשת)
wget https://mirror.openshift.com/pub/openshift-v4/clients/ocp/latest/openshift-install-linux.tar.gz
tar -xvf openshift-install-linux.tar.gz

# הורדת כלי הניהול oc CLI
wget https://mirror.openshift.com/pub/openshift-v4/clients/ocp/latest/openshift-client-linux.tar.gz
tar -xvf openshift-client-linux.tar.gz

# הוספת הכלים ל-PATH (אופציונלי אך מומלץ)
sudo mv oc kubectl openshift-install /usr/local/bin/

שלב 3: יצירת מפתח SSH לגישה ל-Nodes

יש לייצר מפתח SSH שישמש אותנו להתחברות לשרתי ה-CoreOS שיוקמו ב-AWS לצורכי דיבוג:

# יצירת המפתח (ללא סיסמה על מנת להקל על ההתקנה, ניתן לשנות בהתאם למדיניות האבטחה)
ssh-keygen -t ed25519 -N '' -f ~/.ssh/id_rsa

# הפעלת ה-ssh-agent והוספת המפתח
eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_rsa

שלב 4: יצירת קובץ התצורה (install-config.yaml)

בשלב זה נייצר את קובץ הקונפיגורציה. הכלי ישאל אותך מספר שאלות בטרמינל (Cloud Provider, אזור ב-AWS, שם הדומיין ב-Route53, וה-Pull Secret):

openshift-install create install-config --dir=~/openshift-install-dir
חשוב: ניתן לערוך את קובץ ה-install-config.yaml שנוצר בתיקייה בשלב זה אם תרצה לשנות סוגי מכונות (למשל מ-m5.xlarge למשהו אחר) או מספר Worker Nodes. ברגע שמריצים את ההתקנה, הקובץ נמחק, לכן מומלץ לגבות אותו:

cp ~/openshift-install-dir/install-config.yaml ~/openshift-install-dir/install-config.yaml.backup

שלב 5: הרצת ההתקנה ויצירת הקלאסטר

כעת ניתן להתחיל את הפריסה ב-AWS. התהליך לוקח לרוב בין 30 ל-45 דקות:

openshift-install create cluster --dir=~/openshift-install-dir --log-level=info

בסיום ההתקנה, יודפסו למסך כתובת ה-Console, שם המשתמש (kubeadmin) והסיסמה הזמנית שלו.

שלב 6: התחברות לקלאסטר ואימות

ייצא את משתנה הסביבה כדי להשתמש ב-oc מול הקלאסטר החדש:

export KUBECONFIG=~/openshift-install-dir/auth/kubeconfig

# בדיקת מצב ה-Nodes
oc get nodes

# בדיקת מצב ה-Operators
oc get clusteroperators

מחיקת הקלאסטר (Cleanup)

בסיום העבודה או במקרה של תקלה, ניתן למחוק את כל המשאבים שנוצרו ב-AWS בצורה נקייה על ידי הפקודה:

openshift-install destroy cluster --dir=~/openshift-install-dir --log-level=info
