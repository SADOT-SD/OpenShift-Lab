OpenShift 4 Installation Guide (AWS IPI Method)

This guide details the complete process for deploying a self-managed OpenShift Container Platform on AWS using the Installer Provisioned Infrastructure (IPI) method, triggered from a dedicated Linux Bastion host.

Prerequisites

  • A dedicated Linux VM (EC2 instance) running in AWS.
  • AWS IAM User with AdministratorAccess and Long-Term Credentials (Access Key ID & Secret Access Key).
  • A registered Public Domain in AWS Route 53 (Public Hosted Zone).
  • A Red Hat Developer account.
  • Minimum 3 available Elastic IPs in your target AWS region.

Step 1: Prepare the Linux Bastion Host

Connect to your Linux VM via SSH and set up the required authentication.

1.1 Configure AWS CLI

aws configure

Provide your Long-Term Access Key ID, Secret Access Key, target region (e.g., us-east-1), and output format (json).

1.2 Generate and Load SSH Key

This key is injected into the cluster nodes for debugging access.

ssh-keygen -t ed25519 -N '' -f ~/.ssh/id_ed25519
eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_ed25519

Step 2: Download Binaries and Pull Secret

2.1 Create Workspace and Download Tools

mkdir ~/openshift-cluster
cd ~/openshift-cluster
wget https://mirror.openshift.com/pub/openshift-v4/clients/ocp/latest/openshift-install-linux.tar.gz
tar -xvf openshift-install-linux.tar.gz

2.2 Save the Pull Secret

Obtain your pull secret from the Red Hat Hybrid Cloud Console.

nano pull-secret.txt

Paste the JSON token, save, and exit (Ctrl+O, Enter, Ctrl+X).

Step 3: Generate the Cluster Configuration

3.1 Run the Interactive Wizard

mkdir ocp-install
cat pull-secret.txt # Copy the output to your clipboard
./openshift-install create install-config --dir=ocp-install

Answer the prompts:

  • SSH Public Key: /home/ec2-user/.ssh/id_ed25519.pub
  • Platform: aws
  • Region: us-east-1
  • Base Domain: Select your Route 53 domain.
  • Cluster Name: e.g., rad-os
  • Pull Secret: Paste the copied JSON string.

3.2 Backup the Configuration (Crucial)

Architect's Note: The installer consumes and deletes the configuration file during installation. Always create a backup.
cp ocp-install/install-config.yaml ./install-config-clean.yaml

Step 4: Deploy the Cluster

Execute the final build command. The automated process takes approximately 40-45 minutes.

./openshift-install create cluster --dir=ocp-install --log-level=info

Once complete, the terminal will output the Web Console URL and the kubeadmin credentials.

Step 5: Post-Installation Local Client Setup (Windows)

If your Linux VM has incompatible C libraries (e.g., glibc errors), manage the cluster locally from your Windows machine.

  1. Print the kubeconfig on the Linux VM: cat ~/openshift-cluster/ocp-install/auth/kubeconfig
  2. Copy the output and save it locally on Windows (e.g., C:\OpenShift\kubeconfig).
  3. Download the Windows oc.exe client from the OpenShift mirror.
  4. Open Command Prompt and set the environment variable: set KUBECONFIG=C:\OpenShift\kubeconfig
  5. Verify access: oc get nodes
